Install GhosttySet up Windows Terminal

1

Download and install Ghostty

Ghostty is a fast, native terminal app. It handles long output smoothly (which matters when Claude is generating code) and stays out of your way. It's what we'll use in the workshop.

Go to ghostty.org and download the macOS app. Open the .dmg file and drag Ghostty to your Applications folder, just like any other Mac app.

Open Ghostty from your Applications folder (or use Spotlight: Cmd + Space, type "Ghostty", press Enter). You should see a terminal window with a blinking cursor.

Why Ghostty? Claude Code produces a lot of output — code diffs, build logs, agent progress. Ghostty handles this without lagging. The default macOS Terminal app tends to struggle with long sessions. If you already use iTerm2 or another terminal you're comfortable with, that works too.
1

Open Windows Terminal

Windows Terminal is the modern terminal app from Microsoft. On Windows 11, it's pre-installed. On Windows 10, grab it free from the Microsoft Store.

Open it by pressing Win + X and selecting Terminal, or search for "Terminal" in the Start menu. Make sure you're in a PowerShell tab (the default), not Command Prompt.

PowerShell, not CMD Claude Code requires PowerShell. If you see C:\> you're in Command Prompt — open a new PowerShell tab instead. The prompt should show PS C:\>.

Install HomebrewInstall Git for Windows

2

Install Homebrew (the package manager)

Homebrew is like an app store for developer tools that runs in the terminal. We'll use it to install a few things. Paste this into Ghostty:

Ghostty — paste this entire line $ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your Mac password (the one you use to log in). Nothing appears on screen as you type — that's normal. Press Enter when done. Follow any "Next steps" it prints at the end.

Already have Homebrew? Run brew --version. If it prints a version number, skip this step.
2

Install Git for Windows

Claude Code uses Git internally, so this is required. Download and install it from git-scm.com/downloads/win. Use the default settings during installation.

After installing, close and reopen Windows Terminal so it picks up the new PATH. Then verify:

PowerShell PS> git --version # Should show something like: git version 2.47.1.windows.1
WinGet is built in Windows 10/11 comes with WinGet, a package manager similar to Homebrew. We'll use it to install the GitHub CLI later. No setup needed — it's already there.

Getting comfortable with the terminal

If you've never used a terminal before, here's the crash course. The terminal is a text-based way to talk to your computer. Instead of clicking folders in FinderFile Explorer, you type commands.

Good news Commands like pwd, ls, cd, and mkdir all work in PowerShell. The examples below work on both Mac and Windows.
→

Where you are: pwd

"Print working directory." Shows you which folder you're currently in. Think of it as looking at the breadcrumb path in Finder.

GhosttyPowerShell $ pwd # /Users/yourname
→

What's here: ls

"List." Shows what's in the current folder. Like opening a folder in Finder.

GhosttyPowerShell $ ls # Desktop Documents Downloads Projects ...
→

Move around: cd

"Change directory." Navigate into a folder. Use cd .. to go up one level (like clicking the back arrow in Finder).

GhosttyPowerShell # Go into the Documents folder $ cd Documents # Go back up one level $ cd .. # Go to your home folder from anywhere $ cd ~
→

Create folders: mkdir

"Make directory." Creates a new folder. The -p flag creates parent folders if needed.

GhosttyPowerShell $ mkdir -p ~/Projects/workshop
→

The essential pattern

Most of your terminal work with Claude Code follows this pattern:

The workflow you'll use every time # 1. Navigate to your project folder $ cd ~/Projects/my-project # 2. Start Claude Code $ claude # 3. Talk to Claude in plain English "Create a new web page with a hero section and contact form" # 4. Exit when done /exit
Tab completion Press Tab while typing a folder or file name and the terminal will auto-complete it. This saves typing and prevents typos. Try it: type cd Docu then press Tab.

Useful slash commands

While chatting with Claude, you can type commands that start with /. These control the tool itself — they're not messages to Claude.

/

/status — Check your usage

See how much of your rate limit you've used. Useful when Claude seems slow — you might be near your limit.

/

/clear — Start a fresh conversation

Resets the conversation while keeping all project files. Use this when switching between tasks or when Claude seems confused by old context.

/

/help — See all available commands

Shows the full list of slash commands and keyboard shortcuts.

/

/btw — Ask a quick side question

Need a quick answer while Claude is working? Type /btw followed by your question. It answers in an overlay from existing conversation context, doesn't interrupt the main task, and disappears when you press Esc.

/

/review — Code and security review

Run a review of your code or recent changes. Claude checks for common issues — security vulnerabilities, bugs, hardcoded secrets, missing validation.

/

/insights — Analyze your usage

Generates a report on how you've been using Claude Code — patterns, suggestions, and features you might be missing. Good to run after a few sessions.

⎋

Esc Esc — Rewind conversation and code

Press Esc twice to open the rewind menu. Pick a point in the conversation to go back to, and Claude reverts both the chat and the code changes it made after that point. This is your undo button.

  • Restore code and conversation — revert both to that point (most common)
  • Restore conversation only — rewind the chat but keep current code
  • Restore code only — revert files but keep conversation history
Experiment freely Because you can always rewind, feel free to try things. If Claude takes a wrong turn, Esc Esc and you're right back where you started. You can also type /rewind instead.

Understanding permissions

By default, Claude Code asks for permission before doing things like running commands or editing files. You'll see a prompt each time. This is safe but can be slow when you're prototyping.

→

The --dangerously-skip-permissions flag

You'll see this flag throughout the workshop guides. It tells Claude Code to skip all permission prompts — Claude can run commands, edit files, and create things without asking you each time.

GhosttyPowerShell $ claude --dangerously-skip-permissions

The name sounds scary on purpose. Anthropic wants to make sure you only use it intentionally. Here's what it means in practice:

  • Claude can run any terminal command without asking
  • Claude can read and edit any file in your project without asking
  • Claude can install packages, create files, and delete files without asking
When to use it For prototyping and workshop exercises, this flag is fine. It makes the experience much smoother. For production work or repositories with sensitive data, use the default permission mode instead and approve actions individually.
→

All permission modes

Claude Code has five permission modes. The --dangerously-skip-permissions flag is just the most permissive one:

Mode What it does
default Asks for permission on first use of each tool. The safe default.
acceptEdits Auto-approves file edits but still asks for other actions (like running commands).
plan Read-only mode. Claude can analyze code but can't modify files or run commands. Good for reviewing and planning.
dontAsk Auto-denies everything unless you've pre-approved specific tools via /permissions.
bypassPermissions Skips all prompts. Same as --dangerously-skip-permissions.

For the workshop, we'll use --dangerously-skip-permissions to keep things moving. As you get more comfortable, you might prefer default or acceptEdits for everyday work.

→

Set up aliases to save typing

Instead of typing the long flag every time, add shortcuts to your shell config:

Add to ~/.zshrc # Claude Code shortcuts alias cl='claude --dangerously-skip-permissions' alias clc='claude --dangerously-skip-permissions --continue' alias clr='claude --dangerously-skip-permissions --resume'

Run source ~/.zshrc to activate them. Now:

  • cl — start Claude with full permissions
  • clc — continue the last conversation where you left off
  • clr — pick a previous conversation to resume
→

Set up shortcuts to save typing

Instead of typing the long flag every time, add functions to your PowerShell profile. First, open your profile file:

PowerShell PS> notepad $PROFILE

If it asks to create the file, click Yes. Add these lines:

Add to PowerShell profile # Claude Code shortcuts function cl { claude --dangerously-skip-permissions @args } function clc { claude --dangerously-skip-permissions --continue @args } function clr { claude --dangerously-skip-permissions --resume @args }

Save and close Notepad, then reload your profile: . $PROFILE. Now:

  • cl — start Claude with full permissions
  • clc — continue the last conversation where you left off
  • clr — pick a previous conversation to resume

GitHub setup

GitHub is where we'll store and publish code. The CLI lets Claude create repos, push code, and deploy pages without leaving the terminal.

5

Create a GitHub account

If you don't have one already, go to github.com and sign up. A free account is all you need. Takes 2 minutes.

6

Install the GitHub CLI

This lets Claude (and you) interact with GitHub from the terminal — creating repos, pushing code, enabling GitHub Pages.

Ghostty $ brew install gh
PowerShell PS> winget install GitHub.cli

Close and reopen Windows Terminal after installation.

Then log in to your GitHub account:

GhosttyPowerShell $ PS> gh auth login

It will ask a few questions. Choose:

  • GitHub.com (not Enterprise)
  • HTTPS as the protocol
  • Login with a web browser when asked how to authenticate

Follow the prompts — it opens your browser to authorize the connection.

Verify it works:

GhosttyPowerShell $ PS> gh auth status # Should show "Logged in to github.com as yourname"
7

Publish a static page to GitHub Pages

GitHub Pages lets you host static websites for free. Here's the full flow — create a page with Claude, then publish it:

Ghostty # Create a project folder $ mkdir -p ~/Projects/my-page $ cd ~/Projects/my-page # Start Claude $ claude --dangerously-skip-permissions
PowerShell # Create a project folder PS> mkdir -p ~\Projects\my-page PS> cd ~\Projects\my-page # Start Claude PS> claude --dangerously-skip-permissions

Tell Claude to build something and publish it:

You → Claude Build a simple landing page with a hero section, about section, and a contact link. Use clean, modern design. Everything in a single index.html file. Then create a GitHub repository called "my-page", push this project to it, and set up GitHub Pages so it's live on the web. Give me the URL when it's done.

Claude will:

  • Build the HTML page
  • Initialize a Git repository
  • Create a repo on your GitHub account
  • Push the code
  • Enable GitHub Pages
  • Give you the live URL

Your page will be live at https://yourusername.github.io/my-page/ within a minute or two.

Free hosting, forever GitHub Pages is free for public repos. Your page stays online as long as the repo exists. No hosting fees, no server management.
8

Update your page

To make changes later, navigate to the project and start Claude again:

Ghostty $ cd ~/Projects/my-page $ claude --dangerously-skip-permissions
PowerShell PS> cd ~\Projects\my-page PS> claude --dangerously-skip-permissions
You → Claude Update the hero section with a new headline. Then push the changes to GitHub.

Claude edits the file, commits, and pushes. The live page updates automatically within a minute.

Common questions

Can I use a free Claude account?

No. Claude Code requires a paid plan — Pro ($20/month), Max, or Team. Enterprise and API console accounts also work. Check your plan at claude.ai/settings.

Can I use iTerm2 instead of Ghostty?

Yes. Any modern terminal works — iTerm2, Kitty, Warp, or even the built-in macOS Terminal. We recommend Ghostty because it's fast and minimal, but it's not required.

Can I use WSL instead of native Windows?

Yes, but it's not required. Claude Code works natively on Windows via PowerShell. If you already have WSL set up and prefer using it, go ahead — the Mac/Linux instructions will apply. For this workshop, native Windows is simpler.

Is --dangerously-skip-permissions actually dangerous?

For workshop exercises and prototyping, no. It means Claude can run commands and edit files without asking each time. The scary name is intentional — Anthropic wants you to use it deliberately. For production repos or sensitive projects, use the default permission mode instead.

What if Claude Code doesn't install?

Make sure you have a recent macOS version. If the install script fails, try the Homebrew method instead: brew install --cask claude-code. If that fails too, ask for help at the workshop — we'll sort it out.

What if Claude Code doesn't install?

Make sure you're running PowerShell (not CMD). If irm isn't recognized, try: Invoke-RestMethod https://claude.ai/install.ps1 | Invoke-Expression. Also make sure you close and reopen your terminal after installation. If it still fails, ask for help at the workshop.

Do I need to know how to code?

Basic familiarity with the terminal helps (that's why this page covers it). You don't need to know any programming language — Claude writes the code. You describe what you want in plain English.

What if I hit rate limits during the workshop?

Type /status inside Claude Code to check your usage. If you're near the limit, let us know — we can help sort out your access.

Can I use VS Code or Cursor alongside Claude Code?

Absolutely. Claude Code runs in the terminal, your editor runs separately. Use VS Code or Cursor to browse and read the code Claude generates. They complement each other well.

What's the difference between Claude Code and Claude.ai?

Claude.ai is the web chat interface. Claude Code is a command-line tool that runs in your terminal, can read and write files on your machine, run commands, create git commits, and interact with GitHub. Same AI, much more powerful when it can touch your filesystem.
